1. 系统概述

1.1 系统简介

婚恋红娘系统是一个面向婚恋市场的综合管理平台，提供用户注册、资料管理、智能匹配推荐、即时通讯、线下活动管理等功能。系统采用前后端分离架构，前端使用原生HTML/CSS/JavaScript，后端使用Supabase云数据库服务。

1.2 系统架构

层级 技术栈 说明
前端 HTML5 + CSS3 + JavaScript 响应式设计，支持移动端
后端数据库 Supabase (PostgreSQL) 云数据库，提供实时数据订阅
文件存储 Supabase Storage 用户照片、头像存储
认证 自定义认证系统 手机号+密码登录

1.3 用户角色

角色 说明 入口
普通用户 注册用户，使用匹配推荐功能 index.html
中介 婚介机构工作人员，管理客户 agency.html
管理员 系统管理员，全局管理 admin.html

1.4 功能模块

  • 用户模块：注册、登录、资料管理、头像上传
  • 匹配模块：智能匹配算法、匹配结果展示、筛选
  • 消息模块：即时私信、消息通知
  • 活动模块：线下活动发布、报名、管理
  • 公告模块：系统公告发布
  • 管理模块：用户管理、中介管理、数据统计

2. 用户操作手册

2.1 注册与登录

注册流程

  1. 访问系统首页，点击"注册"按钮
  2. 填写手机号（作为唯一账号）
  3. 设置登录密码（建议6位以上）
  4. 输入验证码（短信验证码）
  5. 点击注册完成
注意：手机号将作为登录凭证，请确保真实有效。一个手机号只能注册一个账号。

登录流程

  1. 输入已注册的手机号
  2. 输入登录密码
  3. 点击"登录"按钮
  4. 首次登录需完善个人资料

忘记密码

如忘记密码，请联系客服或管理员重置密码。

2.2 个人资料

基本资料

必填项：姓名、性别、生日、身高、婚姻状况、学历、户口所在地

选填项：体重、职业、收入、房产、车辆、吸烟、饮酒、宗教信仰

个人照片

  • 最多上传9张照片
  • 可设置一张为头像
  • 支持JPG、PNG格式
  • 单张不超过5MB

择偶要求

设置您对另一半的要求，系统将根据要求进行智能匹配：

  • 年龄范围、身高范围
  • 学历要求、婚姻状况
  • 地区要求、体型要求
  • 收入要求、房产车辆
  • 生活习惯（吸烟、饮酒）
  • 宗教信仰

每项要求可设置重要程度：必须满足（红色）、重要（黄色）、一般（绿色）

2.3 匹配推荐

智能匹配原理

系统根据您的个人资料和择偶要求，通过算法计算与其他用户的匹配度：

  • 基础匹配：年龄、身高、学历、地区等基础条件
  • 兴趣匹配：性格爱好、活动偏好的相似度
  • 价值观匹配：婚恋观、消费观的一致性

查看匹配

  1. 进入"匹配推荐"页面
  2. 查看匹配列表，按匹配度排序
  3. 点击用户卡片查看详情
  4. 可对心仪用户"收藏"或"私信"

匹配度说明

匹配度 说明 建议
90%+ 高度匹配 强烈推荐联系
70-90% 较好匹配 建议深入了解
50-70% 一般匹配 可以尝试接触
50%以下 匹配度较低 可作为参考

2.4 私信聊天

发送消息

  1. 在匹配列表点击用户头像进入详情页
  2. 点击"发消息"按钮
  3. 输入消息内容
  4. 点击发送

消息通知

当有新消息时，页面右上角会显示红点提示。未读消息数量会显示在导航栏。

注意：如果对方将您拉黑，则无法发送消息给对方。

2.5 线下活动

查看活动

在活动页面查看所有线下活动，包括：

  • 活动名称、时间、地点
  • 活动内容、费用
  • 已报名人数

报名参加

  1. 选择感兴趣的活动
  2. 点击"立即报名"
  3. 确认报名信息
  4. 等待管理员审核

我的活动

在"我的活动"中可查看已报名的活动状态：报名中、已通过、已拒绝、已取消

3. 管理员手册

3.1 管理员登录

  1. 访问管理后台入口
  2. 输入管理员账号和密码
  3. 点击登录
注意：管理员账号由系统初始化创建，初始账号为 admin / admin123，请首次登录后立即修改密码。

3.2 用户管理

用户列表

查看所有注册用户信息，包括：

  • 基本信息：手机号、姓名、性别、年龄
  • 资料状态：已完善/未完善
  • 账号状态：正常/禁用
  • 注册时间、最近登录时间

用户操作

  • 查看详情：查看完整用户资料和择偶要求
  • 编辑：修改用户基本信息和择偶要求
  • 启用/禁用：控制用户登录状态
  • 删除：永久删除用户（需谨慎）

筛选功能

  • 按状态筛选：全部/正常/禁用
  • 按性别筛选：男/女
  • 按资料完善度筛选
  • 关键词搜索：手机号、姓名

3.3 中介管理

中介列表

查看所有婚介机构账号，包括：

  • 机构名称、联系人、电话
  • 账号状态：正常/禁用
  • 录入客户数量

添加中介

  1. 点击"添加中介"按钮
  2. 填写机构信息
  3. 设置初始密码
  4. 点击保存

中介操作

  • 编辑：修改中介机构信息
  • 重置密码：重置登录密码
  • 启用/禁用：控制账号状态

3.4 公告管理

发布公告

  1. 点击"发布公告"按钮
  2. 填写公告标题和内容
  3. 可选择置顶
  4. 点击发布

公告操作

  • 编辑：修改公告内容
  • 删除：删除公告
  • 置顶/取消置顶：控制公告显示顺序

3.5 活动管理

创建活动

  1. 点击"创建活动"按钮
  2. 填写活动信息：名称、时间、地点、费用、内容
  3. 设置报名截止时间
  4. 点击创建

活动审核

查看用户报名申请，可进行：

  • 通过：允许用户参加活动
  • 拒绝：拒绝用户报名
  • 查看详情：查看报名信息

3.6 数据统计

统计指标

指标 说明
总用户数 所有注册用户数量
活跃用户 近30天有登录的用户
中介数量 合作的婚介机构数量
活动数量 已创建的活动总数
匹配次数 用户查看匹配的总次数

4. 中介手册

4.1 客户管理

录入客户

  1. 点击"添加客户"按钮
  2. 填写客户基本信息
  3. 完善择偶要求
  4. 上传客户照片
  5. 点击保存

客户列表

查看所有录入的客户，包括：

  • 基本信息：姓名、性别、年龄、身高
  • 婚姻状况：未婚/离异/丧偶
  • 资料状态：已完善/待完善
  • 录入时间

客户操作

  • 查看详情：查看完整资料和择偶要求
  • 编辑：修改客户信息
  • 打印：打印客户资料或择偶要求
  • 启用/禁用：控制客户账号状态

4.2 匹配推荐

智能匹配

系统根据客户资料和择偶要求，自动计算匹配度并推荐合适的人选。

匹配结果

  • 按匹配度从高到低排序
  • 显示匹配原因和共同点
  • 可直接联系对方中介

6. API接口

6.1 Supabase REST API

系统使用Supabase提供的REST API进行数据操作，基础URL：

https://awhpfdhxcoycxgfhpfoy.supabase.co/rest/v1/

6.2 主要接口

接口 方法 说明
/users GET/POST/PATCH/DELETE 用户CRUD操作
/users?id=eq.{id} PATCH 更新指定用户
/agencies GET/POST 中介管理
/admins GET/POST 管理员管理
/messages GET/POST 消息收发
/match_results GET 获取匹配结果
/activities GET/POST/PATCH 活动管理

7. 已知问题

🔴 高优先级问题

7.1 安全性问题

问题 描述 状态 修复方案
密码明文存储 数据库中密码未加密存储 待修复 使用bcrypt或类似算法加密

7.2 功能问题

问题 描述 状态
匹配计算函数缺失 match_results表存在但计算函数未正确创建 待修复
触发器引用错误 数据库触发器引用不存在的函数 已禁用
sessionStorage兼容 部分旧版浏览器不支持sessionStorage 低优先级

7.3 数据问题

问题 描述 状态
