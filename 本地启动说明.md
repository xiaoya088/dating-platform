# 婚恋红娘系统 - 本地启动指南

## 问题说明
如果遇到"无法访问网站"的问题，可能是由于：
1. Netlify部署不稳定
2. 网络连接问题
3. 配置文件问题

## 解决方案：本地启动

### 方法一：使用Python（推荐）

```bash
# 进入项目目录
cd /Users/peter/Documents/hongniang

# 给启动脚本添加执行权限
chmod +x start.sh

# 启动服务器
./start.sh
```

或者直接运行：

```bash
# Python3
python3 -m http.server 8000

# 或者Python2
python -m http.server 8000
```

### 方法二：使用Node.js

```bash
# 使用serve
npx serve -p 8000

# 或者使用http-server
npx http-server -p 8000
```

### 方法三：使用VS Code Live Server

1. 在VS Code中安装"Live Server"扩展
2. 右键点击index.html
3. 选择"Open with Live Server"

## 访问地址

启动成功后，在浏览器中访问：

```
http://localhost:8000
```

或者：

```
http://127.0.0.1:8000
```

## 常见问题

### 1. 端口被占用
如果8000端口被占用，可以更换端口：

```bash
python3 -m http.server 8080
```

然后访问：`http://localhost:8080`

### 2. Python未安装
在macOS上安装Python：

```bash
# 使用Homebrew
brew install python3
```

### 3. 数据库连接问题
确保Supabase配置正确，检查`supabase-config.js`文件。

## 推荐工作流程

1. **开发阶段**：使用本地服务器
2. **测试阶段**：部署到Netlify测试
3. **生产环境**：使用稳定的服务器

## 自动化启动

可以创建一个alias，方便快速启动：

```bash
# 在~/.zshrc或~/.bash_profile中添加
alias hongniang='cd /Users/peter/Documents/hongniang && python3 -m http.server 8000'

# 重新加载配置
source ~/.zshrc

# 使用
hongniang
```

## 注意事项

- 本地服务器仅用于开发测试
- 生产环境建议使用专业的Web服务器
- 确保防火墙允许8000端口访问
- 不要在公共网络中暴露本地服务器
